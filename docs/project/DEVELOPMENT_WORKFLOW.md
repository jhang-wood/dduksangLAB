## 🔧 Git 커밋 메시지 규칙

### 커밋 타입
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 스타일 변경 (포맷팅 등)
- `refactor`: 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 빌드 프로세스, 패키지 매니저 설정 등

### 커밋 메시지 형식
```
<타입>: <간단한 요약>

<상세 설명>
- 변경사항 1
- 변경사항 2
- 테스트 결과

🤖 Generated with [Claude Code](https://claude.ai/code)

Co-Authored-By: Claude <noreply@anthropic.com>
```
